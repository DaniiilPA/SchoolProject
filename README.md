Система мониторинга благополучия пользователей с автоматическими SOS-уведомлениями. Проект реализован на микросервисной архитектуре.

## Архитектура системы
* **Bot Service (Aiogram 3):** Клиентский интерфейс для взаимодействия с пользователем.
* **Backend API (FastAPI):** Ядро системы, обрабатывающее логику, настройки и регистрацию.
* **Database (PostgreSQL):** Хранилище данных пользователей, логов и контактов для ЧП.
* **Scheduler (APScheduler):** Фоновый процесс, отслеживающий интервалы "чекинов" и инициирующий тревогу.

## Ключевые технические решения
* **Асинхронность:** Полностью асинхронный стек (FastAPI, SQLAlchemy async, Aiogram).
* **Контейнеризация:** Весь проект разворачивается через Docker Compose (3 сервиса + БД).
* **Безопасность:** Вынос конфиденциальных данных в `.env`, валидация данных через Pydantic.
* **Логика DND:** Учет часовых поясов и периодов сна пользователя при отправке уведомлений.

## Стек
Python 3.10, FastAPI, SQLAlchemy (Async), PostgreSQL, Aiogram 3, Docker, APScheduler, HTTPX.
